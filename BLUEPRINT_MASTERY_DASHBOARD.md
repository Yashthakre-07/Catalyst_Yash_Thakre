## 2. Core Data Models (Pydantic)

The dashboard depends on these two data structures:

```python
from pydantic import BaseModel
from typing import List, Literal, Optional

class SkillAssessment(BaseModel):
    skill_name: str
    required_level: int
    assessed_level: int
    category: Literal["STRONG", "DEVELOPING", "GAP"]
    assessment_reasoning: str
    candidate_feedback: str

class SkillPlan(BaseModel):
    skill_name: str
    category: str
    total_weeks: int
    current_level: int
    target_level: int
    topics: List[dict] # Detailed week-by-week content
```

## 3. The Synthesis Prompt (The "Brain")

This prompt is what turns raw scores into a roadmap. It must be sent to a high-reasoning model (like Gemini Flash).

```text
You are an expert curriculum designer. Based on these Skill Assessments, generate a weekly breakdown.
RULES:
- GAP skills (Score 0-4): Provide 4 weeks of depth.
- DEVELOPING skills (Score 5-7): Provide 2 weeks of focus.
- STRONG skills (Score 8-10): Provide 0 weeks (Mastery).

For each week, provide:
- A clear Objective.
- 'What to Study' list.
- 3 levels of YouTube resources (Channel, URL, and 'Why this video').
- A 'Hands-on Milestone' project.
```

## 5. Implementation Strategy
To achieve "Neural" quality:
1. **Never use static text**: Every summary and description should be generated by the AI based on the candidate's specific answers.
2. **Visual Hierarchy**: Use `st.markdown` with custom CSS to create the "Report Card" feel. 
3. **Downloadable PDF**: Always provide a `pdf_generator` service that mirrors the dashboard data for the candidate to keep.
